# 오늘 학습 정리

## 1. ML 분류 알고리즘 전반

정형데이터 분류에서 자주 쓰이는 알고리즘들을 비교했다.

- **로지스틱 회귀**: feature를 선형으로 조합해 경계를 긋고, sigmoid로 확률 변환. 빠르고 해석은 쉽지만 복잡한 패턴은 못 잡음.
- **SVM**: 두 그룹 사이의 여백을 최대화하는 경계를 찾음. 적은 데이터에 강하지만 느리고 해석이 어려움.
- **KNN**: 가까운 이웃 K개의 다수결로 판단. 구현은 쉽지만 데이터가 많아지면 느려짐.
- **나이브 베이즈**: 베이즈 정리 기반 확률 계산. feature 간 독립을 가정하는데, 의료데이터처럼 feature가 서로 연관된 경우 정확도가 떨어질 수 있음.
- **랜덤포레스트(배깅)**: 나무를 병렬로 여러 개 만들어 투표/평균. 안정적이고 과적합에 덜 취약함.
- **XGBoost(부스팅)**: 나무를 순차적으로 만들어 이전 나무의 실수를 보완. 정형데이터에서 보통 가장 높은 정확도를 보임.
- **신경망(MLP)**: 가중치와 활성화함수를 여러 층 거쳐 학습. 데이터가 많고 패턴이 복잡할 때 강하지만, 데이터가 적으면 오히려 과적합 위험이 있음.

## 2. 신경망 핵심 수식을 숫자로 풀어보기

- **z = wx + b**: 입력값에 가중치를 곱해 더하는 것. 면접 점수 예시로 "점수 매기기"라고 이해함.
- **Sigmoid**: 어떤 숫자든 0~1 사이 확률로 부드럽게 눌러주는 변환 함수.
- **Binary Cross-Entropy**: 정답과 예측의 차이를 벌점으로 매기는 채점 방식. 맞을수록 벌점이 빠르게 줄고, 틀릴수록 벌점이 커짐.
- **Gradient Descent**: 다트 던지기처럼, 틀린 방향과 정도(미분/기울기)를 보고 가중치를 조금씩 반대 방향으로 조정해나가는 과정.
- **미분의 역할**: "지금 위치에서 어느 방향으로 가면 더 커지고 어느 방향으로 가면 더 작아지는지"를 알려주는 도구. 전체 지형을 한눈에 볼 수 없을 때, 국소적인 기울기만 보고 한 걸음씩 내려가는 원리.
- **e**: 그냥 특별한 상수(2.718...). 미분했을 때 자기 자신이 나오는 성질 때문에 수학 전반에서 자주 쓰임.
- **Precision/Recall**: 병원 100명 예시로 TP/FP/FN을 구분. Precision은 "모델이 예측한 것 중 맞은 비율"(분모: 모델이 양성이라 한 수), Recall은 "진짜 환자 중 모델이 찾아낸 비율"(분모: 실제 환자 수). 의료데이터는 Recall이 특히 중요함(놓치는 것이 더 위험하기 때문).

## 3. RAG와 할루시네이션

- **RAG**: 모델 가중치는 그대로 두고, 질문이 들어오면 관련 문서를 검색해서 참고해 답을 생성하는 방식.
- **파인튜닝**: 모델 가중치 자체를 추가 학습시켜 특정 도메인 지식을 내재화하는 방식. GPU 자원이 많이 필요해 LoRA 같은 경량화 기법이 자주 쓰임.
- **할루시네이션 발생 지점**: 검색 실패로 모델이 추측해서 답함, 검색은 됐지만 잘못 해석함, 문서에 없는 디테일을 모델이 채워 넣음, 여러 문서를 섞어 존재하지 않는 조합을 만듦.
- **대응 방법**: 검색 품질 개선(청크 크기, 임베딩 모델, 재순위화), 프롬프트로 "문서에 없으면 모른다고 답하라" 제약, 출처 표시(citation) 강제, 숫자/사실 검증 단계 추가, RAGAS 같은 정량 평가 프레임워크 활용.


## 4. 김태리 교수님 논문 세 편 깊이 읽기

### HI-DR (AAAI 2025)
의약품 추천 문제에서 두 가지 핵심 아이디어를 제안한 논문.

- **Idea 1 (Health Status-Aware Attention)**: 과거 방문이라도 지금과 건강 상태가 다르면 참고하지 않고, 건강 상태가 비슷하면 다른 환자의 방문이라도 참고함. 즉 "누구의 방문인가"보다 "건강 상태가 비슷한가"가 기준이 됨.
- **Idea 2 (EHR Graph+)**: 두 약이 같이 처방된 적이 있어도, "A 처방 시 B도 처방되는 정도"와 "B 처방 시 A도 처방되는 정도"가 다를 수 있다는 비대칭성을 방향성 있는 그래프로 반영함.
- **구조**: 인코더가 과거 방문들 중 관련 있는 것만 골라(Gumbel-softmax) 환자 표현 벡터를 만들고, 예측기가 약을 하나씩 순차적으로 추천함. 이때 (a) 지금까지 추천된 약들을 보고 다음에 필요한 특징을 추론하는 부분과, (b) 비슷한 환자/방문에서 처방된 약에 점수를 매기는 부분(health status-aware attention)을 결합함.
- **비슷한 환자를 찾는 방법(M3)**: 미리 학습된 별도 모델(VITA)로 모든 환자의 표현 벡터를 사전에 계산해두고, 코사인 유사도로 비슷한 환자를 미리 찾아두는 방식. 실시간 계산(M1)은 너무 느리고, 단순 코드 비교(M2)는 맥락을 놓쳐서 이 방식을 택함. 구조적으로 RAG(검색 후 활용)와 유사한 패턴.
- **EHR Graph+와 DDI 그래프의 결합**: 동시처방 패턴을 반영한 약 벡터와 부작용 위험을 반영한 약 벡터를 빼기 연산으로 합쳐서, "자주 같이 쓰이는 약은 벡터가 비슷해지되, 위험한 조합은 벡터가 멀어지도록" 사전에 학습 과정에서 통합함.
- **성과**: 기존 최고 모델보다 정확도 최대 18.69% 향상, 동시에 부작용 위험(DDI rate)은 실제 처방 기록 수준 이하로 유지함.

### VITA (AAAI 2024)
"Carefully Chosen and Weighted Less Is Better"라는 철학으로, 환자 본인의 과거 방문 중에서도 현재와 관련 있는 것만 신중하게 선별하고(relevant-visit selection), 선별된 방문도 관련성 정도에 따라 가중치를 적절히 조절해서 반영함(target-aware attention). 과거에 너무 의존하면 오늘의 새로운 상황을 놓칠 수 있다는 통찰이 핵심.

### MARIO (CIKM 2022)
멀티미디어 추천 문제에서, 데이터(상호작용)가 많아질수록 이미지·텍스트 같은 서로 다른 모달리티의 임베딩이 고유한 특성을 잃고 서로 닮아가는 현상을 실험으로 발견함.

- **Modality-Aware Attention**: 사용자-아이템 상호작용마다 "이번엔 어떤 모달리티(이미지/텍스트/상호작용)가 더 결정적이었는가"를 따로 계산함. 모달리티별 원본 벡터를 합치기 전 단계까지 보관해야 하고, 영향력을 계산하는 추가 attention 파라미터가 필요해 메모리·연산 비용이 늘어남.
- **Modality-Preserving Decoder**: 합쳐진 임베딩에서 원래 모달리티 정보를 복원할 수 있는지 검증하는 장치를 둬서, 모달리티 고유성이 학습 과정에서 흐려지지 않도록 막음.
- **성과**: 기존 최고 경쟁 모델보다 최대 14.61% 높은 정확도(NDCG 기준)를 보임.

## 5. 논문을 읽으며 짚은 비판적 질문들

- **"만성질환에 같은 약이 반복 처방되면, 약을 바꿔야 하는 거 아닌가?"** → 모델은 실제 처방 기록(ground-truth)을 정답으로 학습하기 때문에, "의사들이 실제로 어떻게 처방해왔는지"를 재현하는 것뿐임. 이게 항상 의학적 최선이라는 보장은 없고, 실제 효과 여부를 판단하는 건 검사 수치와 환자 상태를 보는 의사의 영역임. 데이터 사이언스가 해결할 수 있는 영역과 임상 판단이 필요한 영역의 경계를 확인함.
- **"모델이 급성/만성을 어떻게 구분하는가?"** → 의학적으로 직접 진단하는 게 아니라, 과거 방문 기록에 같은 진단이 반복되는지(만성) 또는 처음 나타나는지(급성)라는 시간적 패턴을 통계적으로 학습해 간접적으로 추론하는 것임.
- **"모달리티별 영향력을 따로 계산하면 벡터가 추가로 더 필요한 거 아닌가?"** → 맞음. 모달리티별 원본 벡터를 합치기 전까지 따로 유지해야 하고, attention 가중치를 계산하는 추가 파라미터도 필요해 메모리·연산 비용이 늘어나는 트레이드오프가 있음.
- **"약물 관계를 미리 학습해두면, 환자별로 매번 계산할 필요가 없는 거 아닌가?"** → 약물 간 관계(EHR Graph+, DDI 그래프)는 모든 환자에게 동일하게 적용되는 고정된 지식이라 미리 한 번 학습해두면 충분하지만, "이 환자에게 무슨 약이 필요한가"는 환자마다 다른 가변적인 정보라서 매번 따로 판단해야 함. 두 가지가 본질적으로 다른 종류의 지식이라 둘 다 필요함.
- **HI-DR이 멀티모달인가?** → 아니다. 진단·처치·약물 코드는 다 구조화된 코드 데이터라는 같은 형태이며, 이미지·텍스트·음성처럼 형태가 다른 데이터를 섞은 게 아니므로 단일 모달에 가까움.
